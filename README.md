# bisect

> *A novel about debugging civilization, written inside its own cosmology.*

When instability threatens the Main branch of reality, archivist Logan Voss coordinates twelve Evaluators across a vast commit tree of alternate civilizations in search of a hidden flaw.

Each world represents a different answer to the same question:

**How should human beings live together?**

Some have optimized for trust. Others for freedom, equality, knowledge, order, love, courage, or transformation. Each civilization has solved a problem. Each has created another. As reports arrive from across the search space, what begins as a technical investigation slowly becomes a journey through the adaptive landscape of human possibility itself.

The deeper the search goes, the harder it becomes to determine whether the flaw lies in any particular civilization—or in the search itself.

---

## About This Repository

This repository contains both the manuscript of *Bisect* and the developmental history of the book.

The story takes place in a universe governed by a form of cosmic version control. Civilizations branch, diverge, merge, and fork across time. The structure of this repository mirrors that cosmology.

Readers are not only able to read the novel.

They can watch it evolve.

Every commit, branch, merge, revision, and abandoned idea becomes part of the archaeological record of the project.

In a sense, *Bisect* itself is being subjected to the same process that its characters use to investigate reality.

## Repository Structure

### `main/`

The canonical branch.

Contains the current compiled manuscript and the most polished version of the story.

If you only want to read the book, start here.

---

### `draft/`

Active development.

Individual chapters are developed here sentence-by-sentence, scene-by-scene, and revision-by-revision before being merged into the manuscript.

This is where the story is discovered.

---

### `notes/`

Research, worldbuilding, outlines, philosophy, character profiles, timelines, civilization reports, and supporting materials.

Think of this as the project's research archive.

Not everything in `notes/` will make it into the final book.

---

### `archive/`

Deprecated drafts, abandoned branches, superseded concepts, and historical versions.

Nothing is ever truly deleted.

Only branched.

## Why This Project Is Interesting

*Bisect* combines several ideas that rarely appear together:

* Philosophical science fiction
* Comparative civilization design
* Git and version-control metaphors
* Mythic archetypes
* Systems thinking
* Open development

The central premise is deceptively simple:

> What if every civilization that failed was optimizing for something good?

Rather than presenting utopias and dystopias, *Bisect* explores tradeoffs, unintended consequences, and the possibility that flourishing emerges not from perfect solutions but from maintaining balance among competing virtues.

The most intriguing aspect of the project is that the development process mirrors the story itself. The novel is being written through branching, experimentation, revision, and merge operations—the same mechanisms that govern reality within the narrative.

## Participate in the Search

*Bisect* is not only a novel. It is an ongoing investigation.

Readers are encouraged to engage with the project at whatever level interests them.

### Read

Follow the manuscript as it evolves.

### Comment

Open an Issue with observations, critiques, questions, contradictions, or ideas.

### Fork

Create your own branch of the search.

Explore alternate interpretations, alternate worlds, alternate endings, or entirely new evaluator reports.

### Pull Requests

Contributions are welcome, including:

* Copy edits
* Continuity fixes
* Research
* Worldbuilding suggestions
* Alternative chapter drafts
* Structural feedback
* Philosophical challenges

### Explore the Commit History

The commit history is part of the project.

Browse old commits, compare revisions, inspect abandoned branches, and watch ideas evolve over time.

The final manuscript is only one branch.

The path that produced it matters too.
